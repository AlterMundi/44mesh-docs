# Data Model

This page defines the data structures used for sensor observations, capabilities, and events in the 44Mesh AI agent ecosystem.

---

## Observation

An observation is a single reading from a sensor at a specific time and location.

```json
{
  "id": "obs_7f3a1b2c",
  "timestamp": "2024-03-15T14:30:00Z",
  "node_id": "a1b2c3d4e5",
  "node_ip": "44.30.127.42",
  "sensor_type": "temperature_sensor",
  "value": 23.4,
  "unit": "celsius",
  "confidence": 0.97,
  "location": {
    "lat": -34.6037,
    "lon": -58.3816,
    "alt_meters": 15.0,
    "description": "Buenos Aires, AR"
  },
  "metadata": {}
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique observation identifier |
| `timestamp` | ISO 8601 | Yes | When the reading was taken (UTC) |
| `node_id` | string | Yes | ZeroTier node ID of the reporting node |
| `node_ip` | string | Yes | Public mesh IP of the reporting node |
| `sensor_type` | string | Yes | Capability type (see [Discovery](agent-discovery.md)) |
| `value` | number or object | Yes | The measured value |
| `unit` | string | Yes | Unit of measurement |
| `confidence` | float [0,1] | No | Confidence in the reading (1.0 = certain) |
| `location` | object | No | Geographic coordinates and description |
| `metadata` | object | No | Sensor-specific additional data |

### Value Types

Simple scalar:
```json
{"value": 23.4, "unit": "celsius"}
```

Vector (e.g., wind):
```json
{"value": {"speed": 15.2, "direction_degrees": 270}, "unit": "m/s"}
```

Complex (e.g., spectrum):
```json
{"value": {"frequencies": [...], "powers_dbm": [...]}, "unit": "dBm"}
```

## Capability

A capability describes what a node can measure or do.

```json
{
  "type": "temperature_sensor",
  "unit": "celsius",
  "range_min": -40,
  "range_max": 85,
  "accuracy": 0.5,
  "update_interval_seconds": 60,
  "history_available": true,
  "history_retention_days": 30
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Capability type identifier |
| `unit` | string | No | Default unit for observations |
| `range_min` | number | No | Minimum measurable value |
| `range_max` | number | No | Maximum measurable value |
| `accuracy` | number | No | Measurement accuracy (in units) |
| `update_interval_seconds` | integer | No | How often readings are taken |
| `history_available` | boolean | No | Whether historical data is queryable |
| `history_retention_days` | integer | No | How far back history goes |

## Node Descriptor

Full node description returned by `/capabilities`:

```json
{
  "node_id": "a1b2c3d4e5",
  "node_ip": "44.30.127.42",
  "hostname": "weather-node-ba",
  "location": {
    "lat": -34.6037,
    "lon": -58.3816,
    "alt_meters": 15.0,
    "description": "Buenos Aires, AR"
  },
  "capabilities": [
    {
      "type": "temperature_sensor",
      "unit": "celsius",
      "update_interval_seconds": 60,
      "history_available": true
    },
    {
      "type": "humidity_sensor",
      "unit": "percent_rh",
      "update_interval_seconds": 60,
      "history_available": true
    }
  ],
  "api_version": "1",
  "observations_endpoint": "/observations",
  "uptime_seconds": 864000,
  "last_seen": "2024-03-15T14:30:00Z"
}
```

## Observation Collection Response

The `/observations` endpoint returns a collection:

```json
{
  "node_id": "a1b2c3d4e5",
  "node_ip": "44.30.127.42",
  "count": 3,
  "observations": [
    {
      "id": "obs_001",
      "timestamp": "2024-03-15T14:30:00Z",
      "sensor_type": "temperature_sensor",
      "value": 23.4,
      "unit": "celsius",
      "confidence": 0.97,
      "location": {"lat": -34.6037, "lon": -58.3816}
    },
    {
      "id": "obs_002",
      "timestamp": "2024-03-15T14:29:00Z",
      "sensor_type": "temperature_sensor",
      "value": 23.2,
      "unit": "celsius",
      "confidence": 0.97,
      "location": {"lat": -34.6037, "lon": -58.3816}
    }
  ],
  "pagination": {
    "limit": 100,
    "offset": 0,
    "total": 1440
  }
}
```

## Alert

Alerts are generated by agents when anomalies are detected:

```json
{
  "id": "alert_abc123",
  "timestamp": "2024-03-15T14:35:00Z",
  "severity": "warning",
  "type": "threshold_exceeded",
  "message": "Temperature above 35°C at 3 consecutive readings",
  "source_node_ip": "44.30.127.42",
  "source_sensor_type": "temperature_sensor",
  "observations": ["obs_001", "obs_002", "obs_003"],
  "agent_id": "anomaly-detector-001",
  "acknowledged": false
}
```

### Severity Levels

| Level | Meaning |
|-------|---------|
| `info` | Informational; no action required |
| `warning` | Attention recommended |
| `critical` | Immediate action required |

---

## Data Format Conventions

- **Timestamps**: Always UTC, ISO 8601 format (`2024-03-15T14:30:00Z`)
- **Coordinates**: Decimal degrees (WGS84)
- **IDs**: Lowercase alphanumeric with optional hyphens/underscores
- **Confidence**: Float between 0.0 (uncertain) and 1.0 (certain)
- **Units**: Use SI units where applicable; use string identifiers (not symbols)
